# Grid and Spacing

## Objetivo
Padronizar grid, respiração visual e spacing system.

## Escala base
Sistema de 8 pontos:
- 8
- 16
- 24
- 40
- 64
- 96
- 128

## Tailwind mapping
Sempre que possível, mapear para tokens utilitários do projeto.

## Grid background
O grid background:
- é parte do sistema visual
- ocupa o viewport inteiro
- não deve obedecer ao container
- pode ser reutilizado em seções e páginas vazias

## Aplicação do grid bg
Aplicar em:
- hero sections
- tutoriais
- rank
- áreas de conteúdo

Exceções:
- header
- footer
- faq da página de afiliados (ou faq global, quando a regra exigir fundo sólido)

## Respiração visual
- seções escuras devem respirar
- não colar cards no limite do viewport
- preservar respiro entre título, descrição, CTAs e grid

## Regra de consistência
O spacing deve ser previsível entre:
- páginas
- seções
- componentes
